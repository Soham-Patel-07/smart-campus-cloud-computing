# Smart Campus Cloud Computing Project - Documentation

## Project Overview

This project is an MSC Computer Science course project in Cloud Computing, focused on implementing a Smart Campus infrastructure using Amazon Web Services (AWS). The project demonstrates practical application of cloud computing concepts through AWS cloud architecture.

## Project Objectives

- Build a cloud-based infrastructure for a Smart Campus environment
- Implement IoT connectivity for campus devices
- Create secure and scalable data storage solutions
- Demonstrate real-time data streaming capabilities
- Ensure compliance with cloud security best practices

## AWS Services Implemented

### 1. AWS IoT Core
- Device connectivity and management
- IoT device setup for campus sensors
- Secure communication between devices and cloud

### 2. Amazon S3 (Simple Storage Service)
- Bucket creation for data storage
- Scalable object storage for campus data
- Cost-effective storage solution

### 3. Amazon RDS (Relational Database Service)
- Implemented RDS for DB2
- Managed database for structured campus data
- High availability and reliability

### 4. Amazon Kinesis Data Streams
- Real-time data streaming
- Data processing pipelines
- Real-time analytics capabilities

### 5. AWS WAF (Web Application Firewall)
- Security implementation
- Protection against web threats
- Application layer security

### 6. Additional Components
- Cloud architecture design
- Cost management and optimization

## AWS Certification

This project was completed as part of the AWS Cloud Computing course certification program provided by the university. The hands-on experience with AWS services demonstrates practical knowledge of cloud computing concepts.

## Architecture

The Smart Campus cloud architecture includes:
- IoT device layer for sensor data collection
- Data ingestion layer using Kinesis
- Storage layer with S3
- Database layer with RDS
- Security layer with WAF

## How It Works

1. **Data Collection**: IoT devices collect data from campus sensors
2. **Data Streaming**: Kinesis Data Streams processes real-time data
3. **Data Storage**: S3 provides scalable storage for raw data
4. **Data Management**: RDS manages structured data and queries
5. **Security**: WAF protects the application from web threats
6. **Monitoring**: Cost tracking and resource optimization

## Cost Management

The project includes cost estimation and tracking to demonstrate cloud resource optimization and budget management.
